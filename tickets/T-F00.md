## Context
Wave 0 bootstrap; this deterministic code ticket consumes the current ticket schema and gate map in `.claude/skills/tdd-swarm/references/ticket-format.md` and `.tdd-swarm/gates.md`, with no upstream ticket. `Week_3_AgentForge.pdf`, the tdd-swarm RED/freeze rules, and the hashes recorded by the new coverage/import reports are authoritative.

## Acceptance Criteria
- **AC-1**: Given a ticket with an unmapped AC or a new test without `spec(<ticket>:<AC>)`, when `.tdd-swarm/spec-lint.sh <ticket> <diff-base>` runs, then it exits 1 and names the missing mapping; a complete mapping exits 0.
- **AC-2**: Given a declared package-layer cycle fixture, when `.tdd-swarm/check-import-cycles.py` runs, then it exits 1 with the cycle; the current approved graph exits 0.
- **AC-3**: Given `.tdd-swarm/coverage-policy.md`, when the wrapper runs, then it requires either an executable coverage command plus base-SHA baseline and rejects regression, or an owner-approved `non-applicable` decision with reason, approver, date, and expiry; absent/expired policy exits 1.
- **AC-4**: Given any mapped local gate failure, when `.tdd-swarm/run-local-gates.sh <ticket> <diff-base>` runs, then it exits non-zero, preserves command output, and never skips the remaining report rows silently.
- **AC-5**: Given all gates green, the wrapper records ticket/base/head, commands, exits, coverage-policy hash, and import-graph hash in `.tdd-swarm/reports/<ticket>-gates.md`.

## Definition of Done
- [ ] Independent Test Agent produced clean criterion-tagged RED and Test Reviewer froze it.
- [ ] `.tdd-swarm/run-local-gates.sh tickets/T-F00.md <DIFF_BASE>` exits 0 and report hashes are retained.
